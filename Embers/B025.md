## Why
- Mandiant/GTIG attributes this campaign to the China-nexus actor UNC6508, whose objective was strategic intelligence collection — the report cites "broad collection aspirations, including sensitive defense intelligence related to national security, Indo-Pacific command operations, artificial intelligence, uncrewed vehicle systems, cyber offensive programs, and medical research." Mining internal information repositories at scale is the core data-harvesting behavior, which is exactly T1213.
- The compromised information repository was a REDCap server — "a web-based software platform designed specifically for building and managing online databases and surveys... commonly used... in the North American medical research community" — not SharePoint. The actor deployed a web shell (`help.php`) and the INFINITERED backdoor, which "captures usernames and passwords submitted via POST requests during the login process" and can "execute arbitrary SQL queries against the database and returns the results." A faithful hunt baselines research-DB access first.
- Credential reuse is the access pattern that makes this an information-repository risk for any org: "More than a year after the initial compromise, UNC6508 used overlapping credentials, harvested from REDCap, to access an administrator account." Once an intruder holds valid (and eventually admin) credentials, bulk repository reads blend into normal activity, so the only durable detection is "this user/role accessed materially more, or different, repositories than their baseline" — a baseline that must exist first, hence Embers, not Flames.
- The report names no single repository product across the full intrusion (research DB, email, and admin-level network access), so parent T1213 — Office Suite + SaaS + Databases — correctly scopes the platform-inclusive baseline rather than over-fitting to one sub-technique. The email-exfil leg (a Google Workspace content-compliance rule that "silently BCC-forwarded" matched mail to an attacker Gmail) is captured as an immediate flag and a cross-reference, since enumerating repository config is part of the same collection hunt.
