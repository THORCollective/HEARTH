# B016

What does normal virtual machine and hypervisor binary usage look like across endpoints — which hosts legitimately run QEMU, VirtualBox, or VMware Workstation, and what are the expected binary paths, network patterns, and disk image locations?

| B016 | What does normal virtual machine and hypervisor binary usage look like across endpoints — which hosts legitimately run QEMU, VirtualBox, or VMware Workstation, and what are the expected binary paths, network patterns, and disk image locations? | Defense Evasion | Baseline all endpoints running VM binaries (qemu-system-*.exe, VBoxManage.exe, vmware.exe, vmrun.exe) by collecting process creation events over 30 days. ## Why

- Attackers are increasingly deploying lightweight VMs (QEMU with Alpine Linux) as network proxies to tunnel exfiltration traffic, defeating process-level attribution — without a baseline of legitimate VM usage, these deployments blend into noise on hosts where VMs are expected
- The STXRAT case demonstrated QEMU deployed to ProgramData with a minimal Alpine image specifically to proxy rclone WebDAV exfiltration — a baseline would immediately flag this as anomalous since the host was not an approved VM user and the binary path was non-standard
- VM-based evasion is a growing trend because VMs create a network boundary that breaks EDR visibility — by establishing which hosts and paths are authorized for VM operations, defenders can detect unauthorized deployments with high confidence
- Baselining VM disk image locations and sizes helps detect small attack-focused images (like the ~50MB Alpine raw image used in STXRAT) that are inconsistent with legitimate development or testing workloads
