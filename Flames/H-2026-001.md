# H-2026-001

Threat actors are creating scheduled tasks named 'TPMProfiler' that execute qemu-system-x86_64.exe under the SYSTEM account with port forwarding arguments redirecting custom ports (32567, 22022) to port 22 to establish persistent covert access channels via hidden virtual machines on compromised Windows hosts.

## Why
- This technique enables threat actors to maintain long-term persistent access that is invisible to endpoint security controls, as malicious activity occurs entirely within the hidden VM
- The use of scheduled tasks running under SYSTEM privileges ensures the QEMU VM automatically starts and maintains access across system reboots, supporting credential harvesting, lateral movement, and ransomware deployment
- This behavior has been directly linked to GOLD ENCOUNTER's PayoutsKing ransomware operations and CitrixBleed2 exploitation campaigns, representing a critical detection gap as traditional EDR solutions cannot inspect activity within the virtualized environment
- The specific scheduled task name 'TPMProfiler' and port forwarding configuration (32567, 22022 → 22) provide high-fidelity indicators that distinguish malicious QEMU usage from legitimate virtualization activities
