# H108

An adversary is dumping credentials from the LSASS process memory using tools such as Mimikatz, procdump, comsvcs.dll MiniDump, or direct API calls to access cached domain credentials for lateral movement.

| Hunt # | Hypothesis | Tactic | Notes | Tags | Submitter |
|--------|-----------|--------|-------|------|-----------|
| H108 | An adversary is dumping credentials from the LSASS process memory using tools such as Mimikatz, procdump, comsvcs.dll MiniDump, or direct API calls to access cached domain credentials for lateral movement. | Credential Access | Key indicators include suspicious process access to lsass.exe with GrantedAccess values 0x1010, 0x1410, or 0x1FFFFF (Sysmon Event ID 10), unsigned binaries opening handles to LSASS, use of comsvcs.dll MiniDump via rundll32, and procdump targeting lsass.exe by PID or name. | #credential_access #T1003_001 #lsass #credential_dumping #mimikatz | [Lauren Proehl](https://x.com/jotunvillur) |

## Why

- LSASS credential dumping is the most common technique for obtaining domain credentials during an intrusion — in the ActiveMQ-to-LockBit case, attackers dumped LSASS to harvest domain admin credentials that enabled RDP-based lateral movement to file servers and backup infrastructure
- The specific GrantedAccess value 0x1010 (PROCESS_QUERY_LIMITED_INFORMATION | PROCESS_VM_READ) observed in this intrusion is a well-known signature of credential dumping tools and is rarely used by legitimate software accessing LSASS
- Attackers increasingly use living-off-the-land techniques (comsvcs.dll MiniDump, Task Manager dumps) rather than dropping known tools like Mimikatz, requiring detection logic that focuses on the LSASS access pattern rather than specific tool signatures
- Credential Dumping is a prerequisite for nearly all subsequent attack phases — detecting it early can prevent lateral movement, privilege escalation, and ultimately ransomware deployment
