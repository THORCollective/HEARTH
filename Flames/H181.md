# H181

Adversaries (UNC3753 / Luna Moth / Silent Ransom Group) are using the Windows command shell to download a remote monitoring and management (RMM) installer with curl and silently install it via `msiexec /quiet` immediately after a vishing-induced screen-sharing session, establishing hands-on-keyboard remote access without dropping classic malware.

## Why
- The whole point of the curl→`msiexec /quiet` chain is that every component is a signed, legitimate utility — `curl.exe` ships in-box, `msiexec.exe` is a trusted installer, and the RMM payload is a commercially code-signed MSI — so no single artifact looks malicious; the *sequence and source* are the tell, which is exactly what makes a correlation hunt the right tool.
- This actor deploys data-theft extortion with no ransomware and no custom implant, so endpoint AV has almost nothing to fire on. Catching the install chain is often the earliest host-based opportunity to interdict the intrusion before bulk collection and exfil begin.
- The behavior is highly detectable because legitimate software deployment in a managed enterprise almost never looks like an interactive user shelling out to `curl` to pull an MSI from a bare IP and silently installing it — sanctioned RMM arrives via the deployment tooling, not an ad-hoc one-liner during a Teams/Zoom call.
- Anchoring on the curl→msiexec pattern (rather than on specific RMM names) generalizes: the same delivery one-liner catches whichever RMM the operator rotates to next, and the MSI-source allowlist check survives infrastructure churn.
