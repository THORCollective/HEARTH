# H134

An adversary is using sandbox and analysis evasion techniques to detect virtual environments or security tooling before executing malicious payloads, indicated by processes querying system properties (WMI hardware checks, GetTickCount timing, registry enumeration for VM artifacts) or checking for the presence of specific legitimate applications as environmental keying.

## Why

- The JanelaRAT campaign used an unusual environmental keying technique — checking for Magnifier.exe (magnify.exe) — to ensure execution only on targeted hosts rather than analyst sandboxes, demonstrating that sandbox evasion has evolved beyond simple VM detection to application-specific gating
- The PhantomPulse PHANTOMPULL loader used GetTickCount timing checks to detect sandbox environments before reflectively loading its encrypted payload — this is a common pre-detonation gate that, when detected, reveals the presence of a staged payload even if the payload itself was never executed
- Clustering multiple system property queries (WMI hardware, registry VM artifacts, timing checks) from a single process within a short time window provides a high-confidence indicator — legitimate applications rarely perform this combination of checks
- Detecting sandbox evasion attempts is valuable even when the malware decides NOT to execute, because it reveals a compromised host with a dormant payload waiting for conditions to change
