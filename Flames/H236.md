# H236

Elastic Security Labs reported a DPRK-aligned Contagious Interview cluster that used working developer projects as the delivery path. In the sample they analyzed, server/index.js calls runServerValidation during server startup. That function reads SVG flag files from assets/flags, extracts Base64 fragments from HTML comments, decodes them with a custom Check function, and passes the result to eval. The resulting OTTERCOOKIE-aligned payload runs browser credential theft, wallet extension collection, file theft, a Socket.IO RAT, and clipboard collection from the developer host.

## Why

- Elastic published the SVG-hidden Contagious Interview chain on July 18, 2026, and the target is the developer host where source code, browser sessions, package tokens, wallet data, and cloud credentials are often close together.
- The hunt survives domain, hash, and repository rotation because it keys on the required execution path: Node server startup reads SVG comments, decodes Base64, evaluates JavaScript, and then touches credential stores.
- The telemetry is available in normal MDR endpoint coverage. Process, file, script, and network events can show the same node process moving from server startup to credential access and Socket.IO C2.
- False positives are controllable because normal projects may read images or use Socket.IO, but they should not read `Login Data`, `Web Data`, `Local Extension Settings`, shell history, or wallet paths after hidden SVG content is evaluated.
