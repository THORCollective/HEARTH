# H-2025-001

Threat actors are injecting malicious JavaScript code into legitimate websites that uses JSF*ck obfuscation composed primarily of the symbols []+${} to evade detection, check the website referrer, and redirect victims from search engines to malicious URLs serving malware downloads or other harmful content.

## Why
- JSF*ck obfuscation makes the malicious JavaScript code difficult to analyze and detect by defenders
- Redirecting traffic from search engines to malicious URLs enables threat actors to compromise a large number of victims 
- Malicious redirects can lead to malware downloads, phishing, malvertising, and other harmful activities
- This campaign has infected thousands of legitimate websites, indicating a widespread and coordinated attack
