# H199

North Korea-aligned Sapphire Sleet compromised the `ehindero` maintainer account and poisoned 140+ packages across the `mastra` and `@mastra` npm scopes (including `mastra@1.13.1`) by injecting `easy-day-js@^1.11.21` as a dependency. When a developer or CI runner executes `npm install`, the package's **postinstall** hook (`setup.cjs`) runs an obfuscated 4,572-byte dropper that sets `NODE_TLS_REJECT_UNAUTHORIZED=0`, writes XOR-encoded tracking markers (`.pkg_history`, `.pkg_logs`) to the temp directory, beacons to `https://23.254.164.92:8000/...`, then spawns a detached Node.js process and uses PowerShell reflective .NET loading (`[Reflection.Assembly]::Load()`) to steal browser data and enumerate 166 crypto-wallet extensions before persisting via an `HKCU\...\Run` key and `C:\ProgramData\system.bat`. We hypothesize that on developer and build hosts, an npm/node lifecycle script spawning a shell (powershell.exe/cmd.exe), making outbound network connections to non-registry hosts, or dropping executables/scripts during install will surface this compromise — and other npm postinstall abuse — independent of the specific package or rotating C2.

## Why
- Attributed, current tradecraft: Microsoft documents Sapphire Sleet (DPRK-aligned) abusing the npm postinstall hook in 140+ live `mastra`/`@mastra` packages, so the install-time process/network/file behavior is concrete and worth hunting now across any host that ran `npm install`.
- The durable signal is the lifecycle-script behavior, not the IOCs: package names, C2 IPs/domains, and hashes rotate, but `node.exe`/`npm.exe` spawning a shell or beaconing to a non-registry host during install is a high-fidelity anomaly that generalizes across npm supply-chain campaigns (Axios/UNC1069, PhantomRaven, etc.).
- T1195.002 (Compromise Software Supply Chain) is the most accurate primary because the root behavior is initial access via a tampered software distribution; the actual on-host execution rides T1059 (Command and Scripting Interpreter) via JavaScript (`setup.cjs`) and PowerShell, which is the cross-reference.
- Developer and CI/build hosts are high-value, often under-instrumented, and execute installs with network access and credentials/tokens present — making install-time process lineage and egress (Sysmon EID 1/3/11, 4688, Defender DeviceProcessEvents/DeviceNetworkEvents) a strong, low-noise hunt surface.
