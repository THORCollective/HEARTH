# H210

Threat actors are using PowerShell's `Add-Type` cmdlet with inline C# code containing `System.Windows.Forms.Clipboard.SetText()` to programmatically inject malicious Base64-encoded commands into victims' clipboards, bypassing traditional file-based payload delivery mechanisms.

## Why
- ClickFix attacks have evolved to use sophisticated PowerShell techniques that directly manipulate the Windows clipboard through compiled C# code, making detection more challenging than traditional script-based attacks
- This technique enables attackers to deliver multi-stage payloads (including Lumma Stealer and DarkGate) without writing files to disk, evading file-based detection mechanisms
- The use of `Add-Type` with inline C# to access System.Windows.Forms APIs represents a significant evolution in social engineering attacks, as it combines legitimate Windows functionality with user interaction to achieve code execution
- ClickFix campaigns have skyrocketed in the last year and are now used by multiple threat actors including those distributing AsyncRAT, NetSupport, and various information stealers
- Detection of this specific PowerShell pattern can identify compromise early in the attack chain, before secondary payloads are executed
