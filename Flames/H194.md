# H194

ESET documents the Gentlemen group's in-house EDR-killer framework (GentleKiller, 8+ variants) which operates by installing and starting vulnerable or malicious kernel drivers as Windows services (BYOVD) and then issuing `DeviceIoControl` requests to those drivers to terminate security software — GentleKiller alone targets 400+ processes across 48 security products in a periodic kill loop. The abused drivers are signed third-party or game/anti-cheat drivers (`eb.sys`, `nseckrnl.sys` NSecsoft, `GameDriverX64.sys`, `stpm_old.sys`/`stpm_new.sys` Safetica, `dmx.sys` Zemana WatchDog CVE-2022-42045, `360netmon_wfp.sys` Qihoo 360, IObit `IMFForceDelete` CVE-2025-26125, `googleApiUtil64.sys` Baidu BdApi, `ThrottleBlood.sys`, `havoc.sys` Huawei audio) plus custom rootkits (`G11.sys`/PoisonX). Because every one of these variants must materialize the same kernel-mode foothold — a new service under `HKLM\SYSTEM\CurrentControlSet\Services\*` with a `Type` of 1 (kernel driver) pointing at an `ImagePath` in a user-writable or non-standard path, followed by a driver-load event — we hypothesize that hunting the driver-service install + load chain (7045/4697 service install -> Sysmon EID 6 driver load) surfaces GentleKiller regardless of which specific driver is swapped in. Paired with H195 (invalid/revoked driver signatures).

## Why
- Every GentleKiller variant (8+ documented, plus integrated HexKiller/ThrottleBlood/HavocKiller) converges on the same kernel-mode primitive: register a driver as a service and load it. Hunting the service-install -> driver-load chain catches the technique even as operators swap the underlying driver to dodge hash/name blocklists.
- A kernel-mode service (`Type`=1) whose `ImagePath` is a freshly dropped file outside `System32\drivers` is rare in healthy environments and is exactly what BYOVD requires — high-signal, low-volume.
- The events are reliably logged: 7045 in the System log, 4697 with audit policy, and Sysmon EID 6 is low-volume and high-value, so a standing hunt is cheap to run and hard for the adversary to avoid emitting.
- BYOVD is the enabling step before mass security-process termination (400+ processes here), so detecting the load gives defenders a window before EDR blinding completes.
