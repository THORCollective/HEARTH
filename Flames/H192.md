# H192

Before publishing victim data on its leak site, UNC6240 staged and compressed collected data on the PeopleSoft host using `zstd` with `pv` for progress, then tunneled it over SSH to its DLS mirror (`176.120.22.24`). We hypothesize an adversary has run an archive utility — specifically `zstd` (or `tar`/`gzip`) — against directories of harvested PeopleSoft/HR/PII data, producing a large compressed artifact in a staging directory immediately prior to exfiltration. This is the collection step bridging [[H191]] (web-shell access) and exfil.

## Why
- The report gives the exact `pv | zstd -3 -T0 -o exfil.tar.zst` command line, so the hunt can match a precise, rarely-legitimate process-and-pipe pattern rather than generic compression noise.
- `zstd` driven by `pv` for progress is an interactive-operator pattern that almost never appears in automated PeopleSoft backups, making it a strong behavioral discriminator.
- Catching the archive-staging step provides a detection window after collection but before data leaves the network, enabling response before publication on the leak site.
- The resulting artifact and subsequent outbound SSH to the named DLS mirror chain collection directly to exfiltration, supporting confident triage.
