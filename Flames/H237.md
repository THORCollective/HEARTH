# H237

Threat actors are authenticating to Microsoft 365 environments using Azure CLI with previously compromised username and password combinations from past data breaches to bypass multi-factor authentication controls and gain initial access to cloud tenant resources.

## Why
- This technique allows attackers to gain legitimate access to cloud environments while appearing as normal user activity, making detection challenging without proper authentication monitoring
- Successful authentication via Azure CLI with valid credentials provides attackers with immediate access to cloud resources, enabling lateral movement, data exfiltration, and privilege escalation
- This campaign demonstrates a massive scale attack (81 million login attempts, 78 compromised accounts across 64 organizations) exploiting common misconfigurations in Conditional Access Policies and MFA implementations
- The use of previously breached credentials combined with Azure CLI authentication represents a critical detection gap, as many organizations focus MFA enforcement on web-based logins while leaving CLI access paths unprotected
