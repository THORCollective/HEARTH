# H219

CISA added CVE-2026-12569 to KEV after active exploitation of PTC Windchill and FlexPLM. Field Effect describes the issue as unauthenticated remote code execution caused by improper input validation and deserialization of untrusted data. PTC and follow-on reporting point to JSP web shells, the `/Windchill/login/[0-9a-f]{16}.jsp` naming pattern, possible `flst.txt` file-listing artifacts, and the `X-windchill-req:` header. The useful hunting path is the web shell file write followed by the application server service account spawning command interpreters or transfer tools.

## Why

- CISA added CVE-2026-12569 to KEV and Field Effect reported active exploitation against Windchill and FlexPLM, which makes compromise assessment more useful than a patch-status check alone.
- The web shell path pattern, `flst.txt` artifact, and application-server child process behavior survive IP rotation better than the listed C2 addresses, so the hunt does not depend on one short-lived IOC.
- Most environments running Windchill or FlexPLM can collect endpoint file and process telemetry on the application server, and HTTP access logs add the request path needed to tie exploitation to the file write.
- False positives are manageable because ordinary maintenance may touch application files, but it should not create `/Windchill/login/[0-9a-f]{16}.jsp` and then have the same service account spawn shells or transfer tools.
