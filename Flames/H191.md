# H191

Following CVE-2026-35273 exploitation (see [[H190]]), UNC6240 established persistence by writing JSP web shells and XMLDecoder-triggering XML into the PeopleSoft PSEMHUB application web root. We hypothesize an adversary has dropped a `.jsp` web shell under `<PS_CFG_HOME>/webserv/<domain>/applications/peoplesoft/PSEMHUB.war/` (and/or staged files under `PSEMHUB.war/envmetadata/transactions/`), or planted malicious XML in `<docroot>/envmetadata/data/environment/*.xml` that executes via XMLDecoder on application restart, giving persistent RCE on our PIA/WebLogic server.

## Why
- PSEMHUB.war is an Oracle-shipped static application; any newly created `.jsp` or modified XML inside it is almost never legitimate, making file-creation events in that exact path extremely high fidelity.
- The report names the precise persistence paths and the unauthorized `transactions/`, `logs`, `persistantstorage`, `scratchpad` directories, so the hunt can target real locations rather than monitoring the whole filesystem.
- The XMLDecoder XML path is a stealth backdoor that fires on application restart, so a host can be reinfected post-reboot even after a JSP shell is removed — both must be checked.
- A WebLogic Java process spawning recon utilities is a well-established cross-vendor web-shell behavioral signature and ties file-system persistence to live operator interaction.
