# H193

Threat actors are distributing infostealers (Vidar, Lumma) inside fake AI-tool installers (e.g., "Awesome AI Windows Plugin" / `ProFluxeFlowAi-win-Setup.exe`, fake "DeepSeek V4" archives) delivered via malvertising, SEO-poisoned search, and fraudulent GitHub releases. A user runs a code-signed (often a revoked/MSaaS cert) installer named after an AI brand, which gates execution behind a fake CAPTCHA "Continue" button to evade sandboxes, then drops a renamed Python runtime (`pythonw.exe`) and a `LICENSE.txt` decoy into `%AppData%\Local`, executing a Python-based downloader that retrieves a shellcode loader and beacons to a stealer C2. We hypothesize that hunting for AI-brand-named installers whose process tree spawns an interpreter from a user-writable path will surface this chain regardless of rotating hashes/domains.

## Why
- The article documents a repeatable, hash-rotating ecosystem (GPT-5.5, Claude Code, Kimi, Gemma, GrokCLI, Manus AI, FraudGPT) reusing one loader hash, so behavioral hunting on the execution chain outlives any single IOC.
- The drop pattern is specific and uncommon for benign software: a renamed `pythonw.exe` plus `LICENSE.txt` decoy under `%AppData%\Local` driving a downloader — strong, low-noise signal.
- Signed-but-revoked / MSaaS-signed installers (Fox Tempest) defeat reputation checks, so signer-anomaly + path-anomaly pivots catch what AV reputation misses.
- Single campaigns reached 66,000 devices in a day with hours-to-impact timelines, so a standing hunt materially shrinks dwell time on commodity stealer infections.
