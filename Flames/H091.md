## Why

- Socket identified 72+ new malicious Open VSX extensions since Jan 31 2026 — a significant escalation of the ongoing GlassWorm campaign
- The new tactic publishes clean extensions first to pass review then updates them to list GlassWorm-linked packages as dependencies — the trust-then-pivot pattern makes initial detection very difficult
- Concurrent campaign injecting 151+ GitHub repos with invisible Unicode characters encoding malicious payloads — multiple attack surfaces converging on developer tooling
- Data sources: VS Code extension install logs, extension manifest changes (package.json extensionPack/extensionDependencies fields), network connections from VS Code extension host processes
