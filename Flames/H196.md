# H196

The I-SOON-linked, China-nexus actor FishMonger upgraded its SprySOCKS toolset for Windows and establishes persistence by abusing Image File Execution Options (IFEO). ESET observed the SprySOCKS loader registering `%SystemRoot%\Fonts\ApphostRagistreationVerifier.exe` as a debugger for the legitimate Virtual Disk Service (`vds.exe`) by writing the `Debugger` value under `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\vds.exe`, so the actor binary executes whenever `vds.exe` is auto-started at boot. The campaign even ships a kernel driver (RawWNPF) that hooks `CmRegisterCallbackEx` to hide this exact IFEO key from registry enumeration. We hypothesize that hunting for `Debugger`, `GlobalFlag`, or SilentProcessExit `MonitorProcess`/`ReportingMode` values written under the IFEO/SilentProcessExit hives — where the referenced binary lives in an unusual path (e.g., `\Fonts\`, `%TEMP%`, `%AppData%`) and is not a known debugger — will surface this persistence regardless of which target image the actor chooses, and will also catch the broader class of IFEO injection abuse used for both persistence and privilege escalation.

## Why
- ESET attributes this to FishMonger (I-SOON), a state-aligned China-nexus actor whose SprySOCKS Windows port is freshly observed, so this is live, high-value tradecraft worth a standing hunt.
- The persistence write is the durable signal: the target image (`vds.exe`) and debugger path can change, but a `Debugger`/`MonitorProcess` value pointing at a binary outside `System32` is rare and low-noise in most environments.
- The actor deploys a kernel driver specifically to hide this IFEO key from registry enumeration, which means point-in-time registry scans will miss it — write-time telemetry (Sysmon EID 13 / 4657 / DeviceRegistryEvents) is the only reliable detector.
- IFEO injection is dual-use (persistence + privilege escalation) and broadly abused beyond this campaign, so the hunt generalizes well across actors and accessibility-binary hijacks.
