# H224

Check Point Research exposed **Cavern Manticore**, an Iran-linked modular C2 framework used against Israeli government and IT-sector Active Directory environments. While the framework's infrastructure and DLL payloads rotate, its **Cavern Agent HTTP protocol is static**: XOR+Base64-encoded payloads carried with a distinctive custom header and fixed URI set. HEARTH had no coverage for the Data Encoding (T1132.x) family, so this encoded-C2 fingerprint is a clean gap and an unusually low-false-positive network hunt.

## Why

- **Protocol constants outlive infrastructure.** Domains, IPs, and TLS certs churn; the `X-User-token: …00` header, the `/profile` `/gallery` `/socket` URIs, and the `_;;_` / `_,_` delimiters are baked into the agent and persist across samples — so a signature on them keeps paying off.
- **Encoding-plus-constants beats generic base64 detection.** Base64 in HTTP is everywhere; the Cavern-specific header/URI/delimiter combination is not, which turns a noisy idea into a precise one.
- **Network-only visibility.** This hunt fires from proxy / web-gateway metadata alone — valuable in environments where the AD FS/endpoint side of the intrusion is not fully instrumented.
- **Anchors a multi-technique campaign.** The same framework side-loads `uxtheme.dll` and reflectively loads plugin DLLs from `C:\ProgramData\WinDir\`; catching the C2 fingerprint gives a pivot into the host-side T1129 / T1574.002 activity.
