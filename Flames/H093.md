## Why

- Major AV engines have minimal signatures for Nim/Zig/Crystal binaries — APT36 is exploiting this gap at scale using AI-assisted mass production of unique variants
- AI vibe-coding enables rapid generation of functionally unique samples, defeating hash-based detection and exhausting analyst triage capacity
- Behavioral detection (process injection, suspicious parent-child chains, network callbacks) is more reliable than signature matching for these payloads
- Hunt for unusual PE characteristics: Nim runtime strings (`NimMain`, `nimGC`), Zig stdlib artifacts, Crystal runtime patterns; EDR process tree anomalies are the primary signal
- Data sources: EDR telemetry (process creation, file writes), PE metadata analysis, AV scan logs showing low/zero detection rates on new binaries
