## Why
- Guest-to-host escape vulnerabilities like Januscape (CVE-2026-53359) represent catastrophic failures in cloud isolation, allowing a single malicious tenant to compromise the entire physical host and all co-located guest VMs
- This 16-year-old vulnerability affects both Intel and AMD x86 architectures and was successfully exploited as a 0-day in Google kvmCTF, demonstrating real-world weaponization against major cloud providers
- Detection of suspicious kernel module loading within guest VMs is critical for multi-tenant cloud environments (AWS, GCP, Azure) where nested virtualization is enabled, as successful exploitation grants attackers root access to the hypervisor and enables lateral movement to all tenant workloads on the same physical infrastructure
- The technique requires guest root privileges and kernel module insertion, making it highly observable through audit logs, kernel module loading events, and KVM-specific telemetry before the actual escape occurs
