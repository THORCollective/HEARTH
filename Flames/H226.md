# H226

Sekoia's analysis of the **Mimo** intrusion set (exploiting Craft CMS via CVE-2025-32432) documented dual monetization on compromised Linux hosts: XMRig cryptomining and **IPRoyal Pawns proxyware** that enrolls the victim into a residential-proxy network to sell its bandwidth. Proxyjacking (bandwidth hijacking) is a distinct impact technique from cryptomining, with its own telemetry, and HEARTH had no coverage for it. The durable hunting angle is the proxyware execution primitive — an ELF run from a world-writable path with residential-proxy account flags — which generalizes across proxyware families well beyond Mimo.

## Why

- **Proxyjacking is quiet, profitable, and under-hunted.** Unlike cryptomining it does not spike CPU, so it evades resource-based detection; the signal is the proxyware binary and its network behavior, not host load.
- **The command line is self-identifying.** Residential-proxy clients take enrollment flags (`-accept-tos`, `-email`, `-password`) on the command line — a high-fidelity `execve` signature that survives binary renames.
- **Path + parent make it obvious on a server.** A proxyware ELF dropped into `/tmp` or a web-service working directory and launched by `www-data`/`php-fpm` is nothing a legitimate bandwidth-sharing install ever looks like.
- **Portable, evergreen coverage.** The drop → `chmod +x` → `execve` chain and the open-relay network pattern apply to any proxyware family, making this hunt useful long after the Mimo campaign fades.
